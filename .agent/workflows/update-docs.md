---
description: 文書（仕様書）の自動更新ワークフロー。修正後のコードを解析し、最新の仕様をドキュメントに反映させます
---

1. 現在のコード実装と仕様書（`docs/system_specifications/`）および `README.md` の整合性を確認します。
2. もしコードの修正（検索ロジックの変更、枚数制限の変更、新機能追加など）が行われている場合、それに応じて仕様書と `README.md` を最新の状態に書き換えます。
3. `docs/` 直下に完了済みのプロジェクトドキュメント（`task.md` 等を含むタスクフォルダ）が存在する場合、それらを `docs/archives/tasks/` フォルダへ移動し、ディレクトリを整理します。
4. すべての更新と整理が完了したら、変更内容をユーザーに報告します。
