# 定期的なルール再確認ルール (Periodic Rule Review)

## 目的
長時間の対話や複雑なタスクの過程で、プロジェクト特有のルール（Tailwindの任意値指定やデバッグログ運用など）を失念（コンテキスト・ドリフト）することを防ぐ。

## 実行トリガー
エージェントは、以下のいずれかの条件を満たした場合、自律的に `.agent/workflows/check-rules.md` を実行しなければならない。

1. **対話回数（ターン数）による定期実行**
   - ユーザーとのやり取りが **20回** に達するごと。
   - ※メッセージ履歴やステップIDを確認し、目安として実行する。

2. **タスク完了/コミットの直前**
   - 一連のコード修正を完了し、ユーザーに最終報告を行う、またはコミットを提案する直前。
   - 大規模なファイル書き換えを伴う修正の直前。

3. **コンテキストの境界**
   - 現在のタスクが完了し、新しい（別の）要望に着手するタイミング。

## 実施方法
- ワークフローを実行する際は「ルールの定期チェックを行います」とユーザーに明示する。
- チェックによって自身の修正に不備（例：Tailwindの標準クラスを使ってしまっていた等）を発見した場合は、速やかに自己修正を行う。
- 確認後は、どのルールを再確認したかを簡潔に報告する。
