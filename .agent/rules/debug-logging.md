# デバッグログの運用ルール

## 基本方針
開発中のデバッグ効率を維持しつつ、本番環境でのパフォーマンス低下や情報の露出を防ぐため、以下のルールを遵守する。

1. **環境変数による制御**
   - 開発時のみ出力したいログには必ず `import.meta.env.DEV` によるガードを設ける。
   - 例: `if (import.meta.env.DEV) { console.log(...) }`

2. **本番環境での自動削除**
   - 本番ビルド時に `console.log` および `console.debug` が自動的に削除されるよう `vite.config.ts` で設定する。
   - `console.warn` および `console.error` は重要なシステムエラーの捕捉のため削除せず残す。

3. **ログの品質**
   - ログ出力には接頭辞（例: `[Search]`, `[Sync]`）を付け、何に関する出力か明確にする。
   - 不要になった一時的なログは、タスク完了時に削除する。

4. **センシティブな情報**
   - パスワードやアクセストークン等の機密情報は、開発環境であってもログに出力しない。
