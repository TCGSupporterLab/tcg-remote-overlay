# 禁止事項: 未指示の機能追加および仕様変更の禁止

## 基本原則
ユーザーから明示的な指示がない限り、以下の行為を厳禁とする。

1. **未指示の機能追加（良かれと思っての改善を含む）**
   - UIへのバッジ、ラベル、アイコン等の追加
   - 新しいショートカットキーの割り当て
   - 既存動作の「改善」を目的とした独自の変更

2. **未指示のデータ操作・並び替え**
   - 検索結果のソート順の変更
   - データの加工や自動修正
   - デフォルト値の変更

3. **ドキュメントの自動生成**
   - ユーザーの許可なく `task.md`, `implementation_plan.md`, `walkthrough.md` を生成しない（`user_global` ルールの遵守）

3. **指示範囲外のコード修正の禁止**
   - 指示があった修正箇所と関係のない既存コードの書き換え
   - 無関係なファイルのリファクタリングやクリーニング
   - 「ついでに」行う軽微な変更（タイポ修正、スタイル微調整等も含む）

## 遵守事項
- 修正や機能追加を行う際は、必ずユーザーの要求に基づいた最小限の範囲に限定し、無関係なコードには一切触れないこと。
- ユーザーの意図が不明確な場合は、推測で実装せず、必ず質問して確認すること。
- 「良かれと思って」の独自の判断は、プロジェクトの意図を損なう可能性があることを認識し、慎むこと。
