# タスク: ひらがな検索機能の実装とフィルタロジックの改善

## 概要
ホロライブ公式カード検索ツールにおいて、漢字や英語、特定の固有名詞（ユニット名、マスコット名など）をひらがなだけで柔軟に検索できるようにするため、カスタム辞書の導入とデータエンリッチメント、検索エンジンの改修を行った。また、カラーフィルタのロジックを最新のカード定義に合わせて修正した。

## 目的
- ユーザーが「なきり」で「百鬼あやめ」、「あどべんと」で「Advent」を検索できるようにする。
- 公式にふりがなデータがないため、独自に構築したキーワード辞書を用いて高精度な検索を実現する。
- 単色・多色フィルタの挙動を、ユーザーの意図（純粋な単色 / 2色以上の混色）に合わせる。

## 完了した主な項目
1.  **キーワード分析と辞書の構築**:
    *   既存カードデータから出現頻度の高いワードを抽出。
    *   漢字、英語、数字を含むキーワードに限定したひらがな辞書 (`kana-dictionary.json`) の作成。
    *   「泥棒建設」「あん肝」等のWiki由来のワードの追加。
2.  **データ付与スクリプトの実装**:
    *   `fetch-hololive-data.js` を拡張し、スクレイピング時に自動で `kana` フィールドを付与。
    *   既存データ一括更新用の `enrich-cards.js` を作成。
3.  **検索用フックの改修 (`useCardSearch.ts`)**:
    *   検索対象の `_joinedText` に `kana` フィールドを統合。
    *   カラーフィルタ（`multi` / `not_multi`）の判定ロジック修正。
4.  **UIの改善**:
    *   フィルタパネルの文言を「単色」に変更し、分かりやすいUIへ修正。
    *   検索プレースホルダーの調整。
