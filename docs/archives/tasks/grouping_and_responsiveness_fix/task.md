# タスク: グループ化と操作レスポンスの改善

## 概要
ウィジェットをグループ化した際、または複数選択して移動する際の操作レスポンス（特に開始時のラグ）を改善し、表示上の不整合を解消します。

## 課題
1. **操作開始時のラグ**: 操作終了後の同期無視期間（150ms）により、グループ全体を動かそうとした際に一部のウィジェットが遅れて動き出す。
2. **境界枠の重なり**: 複数のグループを選択した際、各グループの枠と全体の枠が重なって表示され、視覚的に混乱を招く。
3. **ステートマッピング不備**: コインウィジェットが誤ったステート（hololive_sp_marker）を参照していた。

## 解決策
1. **Nonceベースのシグナル伝搬**: 親(`App.tsx`)で操作が始まった瞬間にNonce（通知用数値）をインクリメントし、全ウィジェットに即座に同期無視フラグを解除させる。
2. **条件付きレンダリング**: 複数選択時は個別のグループ枠を非表示にし、全体枠のみを表示する。
3. **プロップの修正**: コインウィジェットの `externalState` 参照先を修正。
