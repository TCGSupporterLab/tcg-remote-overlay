# 修正計画 - 画像ダウンロードスクリプトの無駄なリサイズ修正

## 概要
`scripts/download-images.js` において、画像サイズがすでに規定（400x559）であるにもかかわらず、毎回の実行で「Resized」と判定されてしまう問題を修正します。
これは、一時ファイルへの書き出し判定ロジックが不適切なために発生しています。

## 修正内容
1. `processImage` 関数のロジックを修正
   - サイズが一致している場合、かつ入力がファイルパス（string）である場合は、書き出しを行わず `false` を返すようにします。
   - 入力が Buffer（新規ダウンロード）の場合のみ、サイズが一致していても保存するようにします。

## 確認事項
- 修正後、一度実行して「Resized」または「Skipped」になることを確認。
- 再度実行した際に、すべての画像が「Skipped」になることを確認。
