# 修正内容の確認 (Walkthrough)

## 変更の概要
ユーザーの利便性向上のため、アプリケーション起動時の設定メニュー表示を制御できるようにしました。

## 主な変更点

### 1. ストアの更新 (`src/store/useWidgetStore.ts`)
- `WidgetStoreState` の `settings` インターフェースに `hideSettingsOnStart` を追加しました。
- `getInitialState` で `localStorage` からのロードを実装しました。
- `subscribe` ブロックにより、設定変更が自動的に保存されることを確認しました。

### 2. 設定メニュー UI の更新 (`src/components/SettingsMenu.tsx`)
- ヘッダー部分にチェックボックスを追加しました。
- 「アクセス時に非表示」ラベルを配置しました。

### 3. アプリケーション初期化の変更 (`src/App.tsx`)
- `showSettings` の初期状態をストアの設定値から派生させるようにしました。
- `SettingsMenu` に新しい設定値を流し込むためのプロップス定義を更新しました。

## 動作確認手順
1. 設定メニューを開き、上部の「アクセス時に非表示」にチェックを入れます。
2. アプリケーションをリロード（または再起動）します。
3. 設定メニューが表示されず、直接オーバーレイ（またはビデオ選択画面）が表示されることを確認します。
4. 右クリックまたはEscキーで設定メニューが正常に開くことを確認します。
5. チェックを外して再度リロードし、今度はメニューが表示された状態で開始されることを確認します。
