# 実装計画: 網羅的仕様書の再構築

## 1. 背景と目的
これまでの修正に伴いドキュメントが分散し、一部古い内容が含まれていたため、現在のコードを正（Truth）とした全機能の仕様書を階層構造で再構築する。

## 2. 実装アプローチ

### 2.1. ディレクトリ構造の設計
`docs/system_specifications/` を起点とし、以下の構成でファイルを分割する。
- `index.md`: 全体の概観
- `core/`: 同期基盤、表示モード
- `features/`: 各ゲームモード別、共通ツールの詳細

### 2.2. 仕様の抽出
- `App.tsx`: グローバルステートとウィンドウ管理、同期の基盤ロジック。
- `useCardSearch.ts`: ホロライブ検索エンジンの詳細（正規化、フィルタ定義）。
- `YugiohTools.tsx`: 遊戯王の計算機ロジック、ショートカット、履歴管理。
- 各コンポーネント: ダイス・コインの挙動、UIデザインのコンセプト。

## 3. 整理ルール
- 以前作成した `card_filter_spec` などの暫定的なフォルダは削除し、新体系に統合する。
- 今後の開発者がドキュメントを読むだけで実装の意図（ショートカット設定や検索の正規化ルールなど）を完全に把握できるようにする。
