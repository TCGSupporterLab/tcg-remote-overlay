# フィルタ表示順制御 (__order 対応) の実装完了

`metadata.json` 内の `__order` キーに基づき、フィルタタブと項目の表示順を自由に制御できる機能を実装しました。

## 実施内容

### 1. メタデータ定義の追加
- **[generate-metadata.js](file:///f:/dev/github/tcg-remote-overlay/scripts/generate-metadata.js)**: 
  - `metadata.json` の先頭に `__order` オブジェクトを出力するようにしました。
  - 現在、以下の順序がデフォルトとして定義されています：
    - **色**: 白, 緑, 赤, 青, 紫, 黄, 無, 多色
    - **タイプ**: 推しホロメン, ホロメン, Buzzホロメン, サポート, ファン, マスコット, ツール, イベント, スタッフ, LIMITED, エール
    - **Bloom**: Debut, 1st, 2nd, Spot

### 2. ソートロジックの実装
- **[useCardSearch.ts](file:///f:/dev/github/tcg-remote-overlay/src/hooks/useCardSearch.ts)**:
  - **カテゴリ（タブ）のソート**: `__order` のキー出現順を優先し、定義されていないカテゴリは末尾に昇順で表示されます。
  - **項目（ボタン）のソート**: 各カテゴリ内で `__order[category]` の定義順を優先し、定義外の項目は末尾に昇順で表示されます。
  - **クリーンアップ**: `__order` キー自体はフィルタリングの対象外（`excludeKeys`）に設定しました。

## 動作確認事項
- [x] フィルタタブの並びが「色」→「タイプ」→「Bloom」の順になっている。
- [x] 「タイプ」タブ内のボタンが、指定通り「推しホロメン」から始まる順序になっている。
- [x] 指定外の項目（もしあれば）が、指定項目の後にアルファベット/五十音順で並んでいる。

---
これにより、カードの追加やメタデータの変更があっても、意図した通りの使いやすいフィルタ順序を維持できるようになりました。
