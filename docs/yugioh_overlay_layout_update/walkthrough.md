# 遊戯王OCGオーバーレイのレイアウト変更確認

遊戯王OCG側のオーバーレイ画面の視認性と操作性を向上させるため、レイアウトの刷新と不要なUI要素の削除を行いました。

## 修正内容のハイライト

### 1. オーバーレイ画面の縦型レイアウトへの移行
- 従来の横一列配置から、対面対戦（リモートデュエル）を意識した上下配置に変更しました。
- **配置順（上から）**:
    1. **Player 2**: 相手側の視点として**常に180度回転**した状態で表示されます。
    2. **ダイス・コイン**: 中央に配置し、両プレイヤーから見やすい位置に固定しました。
        - **レイアウト調整**: オーバーレイ時のみ、視認性向上のためコインを少し右にずらして配置しています。
    3. **Player 1**: 自分のライフポイントを画面下部に表示します。
- **ショートカットキーの厳格な分離**: 表示レイアウトに合わせて、各画面で使用可能なキーを限定しました。
    - **オーバーレイ画面 (縦長)**: **上下キー (`↑`/`↓`) のみ**機能。左右キーは無効。
        - **上キー (`↑`)**: 画面上部の **Player 2** を選択
        - **下キー (`↓`)**: 画面下部の **Player 1** を選択
    - **操作画面 (コントローラー)**: **左右キー (`←`/`→`) のみ**機能。上下キーは無効。
        - **左キー (`←`)**: 画面左側の **Player 1** を選択
        - **右キー (`→`)**: 画面右側の **Player 2** を選択

### 2. オーバーレイウィンドウの最適化とパーソナライズ
- **初期サイズの調整**: 遊戯王OCGは **350x500**、ホロライブOCGは **400x700** をデフォルトとして設定しました。
- **サイズの記憶（永続化）**: 
    - ユーザーがオーバーレイウィンドウのサイズを任意に変更すると、そのサイズが**ゲームモードごとにブラウザに保存**されます。
    - 次回「オーバーレイ起動」をクリックした際、前回閉じた時と同じサイズで自動的に開くようになります。
    - 遊戯王とホロライブで個別にサイズを記憶するため、用途に合わせた最適なレイアウトを維持できます。
- **内部要素の超コンパクト化**: 各カードのパディングや要素間のギャップを極限まで削減し、最小限のウィンドウスペースで全ての情報が表示されるように調整しました。

### 2. 操作画面（非オーバーレイ）のUI整理
- 各プレイヤーカードの右上にあった**回転ピン（回転トグルボタン）を削除**しました。
- これにより、操作画面がよりシンプルになり、誤操作を防ぐことができます。

### 3. コードのクリーンアップ
- UIの変更に伴い使用されなくなった以下の要素を削除しました：
    - `lucide-react` からの `RotateCcw` アイコンのインポート
    - 回転状態をトグルする `toggleRotation` 関数
    - 旧レイアウト用の `overlayRowClass` スタイル変数

## 確認手順
1. アプリケーションを起動し、遊戯王OCGツールを表示します。
2. 操作画面で、プレイヤーカードの右上に回転ボタンが表示されていないことを確認します。
3. オーバーレイモードを有効にし、ライフポイントとダイス/コインが縦に並んでいることを確認します。
4. 最上部の Player 2 の表示が180度反転していることを確認します。
