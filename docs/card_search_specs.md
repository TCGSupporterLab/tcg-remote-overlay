# ホロライブOCG カード検索機能 仕様書

## 概要
ホロライブOCGのカードデータを検索・閲覧し、対戦中に参照するための機能。
現在はUIレイアウトの再構築待ちのため、機能自体を一時的に無効化している。

## データソース
- **データファイル**: `src/data/hololive-cards.json`
- **画像リソース**: `public/images/cards/` (ローカル保存済み)
- **更新スクリプト**: `npm run update-data` (Fetch + Download)

## 機能要件

### 1. 検索・フィルタリング
- **キーワード検索**: カード名、No（ID）、テキストなどを対象に検索。
- **絞り込みフィルタ**:
    - **カードタイプ**: 推しホロメン、ホロメン、サポート、エール、その他
    - **色**: 白、緑、赤、青、紫、無色
    - **レアリティ**: RR, R, C, Uなど
    - **HP**: 数値範囲指定（以上/以下）
    - **タグ**: ターゲットあり、Bloom可、など（特性ベース）

### 2. 詳細表示
- 選択したカードの拡大画像を表示。
- カードの基本情報（ID, レアリティ等）を表示。

### 3. ピン留め機能（お気に入り）
- **機能**: よく使うカードや現在プレイ中のカードを一時保存する。
- **保存数**: 最大10枚。
- **永続化**: `localStorage` に保存し、リロードしても保持する。
- **操作**:
    - 詳細画面から「ピン留め」ボタンで追加/解除。
    - ピン留めエリアから「削除」ボタンで解除。
    - 「全て解除」ボタンで一括削除。

## 既知の課題（再実装時の注意点）
- **レイアウト**:
    - `flex` や `grid` を用いた際、ブラウザの高さ計算（`h-full`, `overflow`）が複雑になり、スクロールバーが表示されなかったり、画面全体がスクロールしてしまう不具合が多発した。
    - 再実装時は **「親コンテナの高さ固定」** と **「スクロールエリアの独立」** を最優先に設計する必要がある。
