# 修正内容の確認 (Walkthrough)

## 変更の概要
SPマーカーの位置調整と、カード表示の角丸処理を刷新しました。これにより、OBS等でのオーバーレイ表示時にマーカーが見切れる問題が解消され、かつカード画像の四隅を綺麗に表示できるようになりました。

### 主な変更点

#### 1. `HololiveTools.tsx`
- **オフセットの個別化**: `IMAGE_FOLLOW_OFFSET` と `TEXT_FOLLOW_OFFSET` を導入。
- **角丸定数の導入**: `OVERLAY_CARD_RADIUS`（デフォルト `17px`）を定義し、エクスポート。
- **レイアウト修正**: SPマーカーをクリッピング領域の外に移動し、自由にはみ出して表示できるように変更。

#### 2. `CardSearchContainer.tsx`
- 操作画面の左側詳細表示エリアに `OVERLAY_CARD_RADIUS` を適用。
- オーバーレイ画面と同じスタイルのトリミングを行うことで、操作系・表示系の見た目を一致させた。

#### 3. `App.css`
- `.overlay-card-frame` や `.overlay-text-container` にあった `!important` 付きのボーダー・背景設定を削除。
- これにより、JSコンポーネント側からの条件付きスタイル（画像モードでは枠なし、等）が正しく反映されるようになった。

#### 4. `SPMarkerWidget.tsx`
- ウィジェット自体のサイズ定数を整理（追従モードと独立モードで個別に設定可能化）。

## 検証結果
- [x] テキストモード時、SPマーカーの下側が見切れない。
- [x] 画像モード時、外側の不要なグレー枠が表示されない。
- [x] `OVERLAY_CARD_RADIUS` を変更すると、全ての画面の角丸が連動して変わる。
